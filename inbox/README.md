# inbox/ — moved to profile/inbox/

The inbox is where you drop user-supplied inputs the agents may need to
reference: branded PPTX/XLSX/DOCX masters, prior close packs, vendor
reports, planning workbooks, screenshots, exemplars, anything not yet
ingested.

Because this content is business-specific, the **actual inbox lives at
`../profile/inbox/`** (gitignored). This directory is intentionally a stub.

If you cloned fresh and `../profile/inbox/` doesn't exist, the
[onboarding skill](../.claude/skills/onboarding/SKILL.md) creates it
during first-run setup. Drop your materials there.
